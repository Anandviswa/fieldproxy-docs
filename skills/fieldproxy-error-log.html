Error Log

Every bug the team has hit — date, symptom, root cause, and fix. Learn from others' mistakes.

How to Use

When a generated view has a bug:

  1. Add a row below with the date, view, symptom, root cause, and which skill was updated
  2. If a new rule is needed, add it to the relevant skill file
  3. If an existing rule was insufficient, update it with the new edge case

Tip: Use Cmd+K to search bugs by symptom, root cause, or project name. Use the filter buttons to narrow by project.
